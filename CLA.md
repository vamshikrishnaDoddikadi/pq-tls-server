# Contributor License Agreement (CLA)

Thank you for contributing to PQ-TLS Server. To ensure that the project can remain open-source while also offering commercial versions in the future, we require all contributors to sign a Contributor License Agreement.

This agreement gives the project maintainer the permission to use your contributions in both the open-source (MIT) version and any future commercial editions, while you retain ownership of your contribution.

## Types of Contributions

### Small Contributions (Trivial Changes)

Typo fixes, documentation clarifications, and minor refactors (less than 10 lines changed) are considered trivial and do not require a signed CLA.

### Substantial Contributions

Any code change, new feature, performance optimization, or bug fix exceeding 10 lines requires a signed CLA.

## How to Sign

1. Read the full CLA text below
2. Add a file named `CLA_SIGNED.txt` to the root of your pull request with the following content:

```
I, [YOUR FULL NAME], agree to the terms of the PQ-TLS Server Contributor License Agreement.
GitHub Username: [YOUR GITHUB USERNAME]
Date: [YYYY-MM-DD]
```

3. The project maintainer will verify this before merging your PR.

Alternatively, you can sign via the CLA Assistant GitHub App (once configured).
